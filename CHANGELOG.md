# Changelog

บันทึกการเปลี่ยนแปลงและการอัปเดตทั้งหมดของ **Golang VOD Module (Standalone HLS/DASH Transmuxer)**

---

## [1.0.1] - 2026-06-11

### ปรับปรุงและแก้ไข (Changed/Fixed)
- **เพิ่มประสิทธิภาพการสแกนและวิเคราะห์ไฟล์ MP4 (Fast Binary MP4 Parsing)**: ปรับปรุงการดึงโครงสร้างอะตอมขนาดใหญ่ (`stts`, `ctts`, `stsz`, `stsc`, `stco`, `co64`, `stss`) ผ่าน Binary Parser เพื่อหลีกเลี่ยง Overhead จากระบบ Reflection ของแพ็กเกจ `go-mp4` ช่วยลดเวลาถอดรหัส Metadata ลงเหลือประมาณ **19ms** (จากเดิม 8–10 วินาทีในไฟล์ขนาดใหญ่)
- **เพิ่มประสิทธิภาพการแบ่งเซกเมนต์ HLS (HLS Segment Generation Optimization)**: ปรับปรุงฟังก์ชันการค้นหาตำแหน่งข้อมูลเสียง (`findAudioSampleIdx`) จากการค้นแบบเชิงเส้น (Linear Scan) มาเป็นการค้นแบบทวิภาค (Binary Search) ทำให้ลดเวลาการจัดกลุ่มและสร้างเซกเมนต์ HLS ลงเหลือเพียง **2ms** (จากเดิม 1.15 วินาที) ทำให้หน้าสารบัญ `video.m3u8` ตอบสนองเร็วขึ้นถึงระดับ **27ms**

## [1.0.0] - 2026-06-10

### เพิ่มเข้ามา (Added)
- **พอร์ตระบบจาก C Module เป็น Pure Goสำเร็จ**: พอร์ตการทำงานหลักด้าน Transmuxing ของ `nginx-vod-module` (Kaltura) มาเป็นภาษา Go 100% โดยไม่ต้องพึ่งพา CGo หรือ FFmpeg ไดนามิกลิงก์
- **Mapped Mode**: รองรับการค้นหาและดึงที่อยู่ไฟล์วิดีโอ MP4 จริงผ่าน HTTP JSON API Upstream (โดยดีฟอลต์ชี้ไปที่พอร์ต `8888`) เพื่อความปลอดภัยสูงสุดตามโครงสร้างของ VdoHide
- **ระบบแคชข้อมูลภาพ/เสียง (In-Memory Metadata Cache)**: ระบบจดจำโครงสร้าง MP4 Box/Atoms และตำแหน่งคีย์เฟรมในแรม ช่วยให้ตอบสนองต่อคำขอ Request เซกเมนต์ถัดๆ ไปได้ทันทีโดยไม่ต้องอ่านไฟล์วิดีโอใหม่ทุกรอบ
- **HLS Streaming (MPEG-TS & fMP4)**:
  - สร้าง Master Playlist (`master.m3u8`) และ Media Playlist สำหรับวิดีโอ/เสียงสตรีมเดี่ยว (`index-v1-a1.m3u8`)
  - ประกอบเซกเมนต์ไฟล์ย่อย MPEG-TS (`seg-{num}-v1-a1.ts`) พร้อมระบบ Interleave สัญญาณภาพและเสียงให้สอดคล้องกัน
  - รองรับ Init Segment และ Media Segments ในฟอร์แมต fMP4
- **DASH Streaming (fMP4)**:
  - สร้าง Manifest ไฟล์แสดงสตรีมหลัก (`manifest.mpd`)
  - ประกอบ Fragmented MP4 สำหรับ Init (`init-v1.mp4`, `init-a1.mp4`) และเซกเมนต์ย่อย (`seg-{num}-v1.m4s`, `seg-{num}-a1.m4s`)
- **การจัดการผ่าน JSON Config**: เพิ่มการโหลดการตั้งค่าระบบผ่านไฟล์ `config.json` (ตั้งค่าพอร์ตทำงาน, ลิงก์ Upstream, และขนาดความยาวของเซกเมนต์เริ่มต้น)
- **ระบบทดสอบความถูกต้อง (Test Coverage)**: เพิ่ม Unit Tests สำหรับเครื่องมือวิเคราะห์ MP4 Parser และ Integration Tests สำหรับการจำลองเว็บเซิร์ฟเวอร์ในการดึงข้อมูล

### ปรับปรุงและแก้ไข (Changed/Fixed)
- **ยกเลิกฟีเจอร์ที่ไม่จำเป็น (Removed Features)**: ถอดส่วนการทำงานสร้างรูปภาพย่อ (Thumbnail / `/thumb/`) ออกชั่วคราว เพื่อลดความยุ่งยากของ Dependencies และเน้นความเสถียรของการส่งสตรีมมิ่งผ่าน HLS/DASH เป็นหลัก
- **แก้ไขข้อผิดพลาดในการรวมสัญญาณ TS (TS Muxing)**: แก้ไขข้อผิดพลาด `astits: PCR PID invalid` โดยกำหนด PCR PID (Program Clock Reference) ให้ตรงกับ PID ของสตรีมภาพ (256) หรือสตรีมเสียง (257) อย่างถูกต้อง ก่อนที่จะทำการสร้างตาราง PAT/PMT ของ MPEG-TS
- **แก้ไขข้อผิดพลาดการแปลงโครงสร้าง DASH (fMP4 Muxing)**: แก้ไขข้อผิดพลาด `CHUNK_DEMUXER_ERROR_APPEND_FAILED` บนเบราว์เซอร์ โดยสร้าง Struct ครอบการทำงานของ Visual/Audio Sample Entry (avc1/mp4a) เพื่อบอกประเภทกล่องแก่ `go-mp4` ส่งผลให้สามารถเขียนรายละเอียดวิดีโอ (ความกว้าง, ความสูง) และรายละเอียดเสียง (Sampling Rate) ลงในหัวข้อมูลเริ่มแรก (Initialization Segment) ได้อย่างสมบูรณ์ และเบราว์เซอร์สามารถถอดรหัสไปเล่นได้ตามปกติ
- **แก้ไขปัญหาการเล่นกระตุกของ DASH (DASH Playback Stuttering)**: ปรับปรุงการคำนวณช่วงเวลาในระบบดัชนีเซกเมนต์ (`SegmentTimeline` ใน `manifest.mpd`) จากเดิมที่ใช้เวลาแสดงผล (PTS) ให้เปลี่ยนมาใช้เวลาถอดรหัส (DTS) เพื่อให้สอดคล้องอย่างสมบูรณ์กับฟิลด์ `baseMediaDecodeTime` ในกล่อง `tfdt` ของแต่ละเซกเมนต์ ช่วยกำจัดช่องว่างช่วงรอยต่อรอยต่อเซกเมนต์ที่เคยทำให้เครื่องเล่นกระตุกและทำการปรับข้ามเวลา (seeking) ซ้ำ ๆ ทุกครั้งที่โหลดเซกเมนต์ใหม่
- **แก้ไขปัญหาเฟรมสลับ/ภาพสั่นกระตุกจากกล่อง trun (DASH Frame Stuttering)**: บังคับการตั้งค่า `trun.Version = 1` (Track Run Box) เพื่อเปิดใช้งานการบันทึกค่าระยะเวลาเบี่ยงเบนในการแสดงผลของภาพ (Sample Composition Time Offsets) สำหรับวิดีโอที่มี B-Frames (ทำให้การเขียนค่าตำแหน่งเฟรม H.264 มีความถูกต้องและไม่สูญหาย บราวเซอร์สามารถจัดเรียงเฟรมถอดรหัสออกมาสตรีมได้ราบรื่นไม่มีการสั่นกระตุก)
